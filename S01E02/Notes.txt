Структуры данных
Время работы для каждой структуры данных измеряется отдельно для каждой операции

Куча: хранит множество элементов, элементы сравнимы между собой, можно вставлять элементы, и можно достать минимальный элемент

Двоичная куча: бинарное дерево.
Если пронумеровать все элементы по уровням и перенести в массив.
то i - элемент, 2i+1 - left child, 2i+2 - right child, родитель  - (i-1)/2
в родителе элемент меньше чем в детях
sift-up - просеивание наверх. Сравниваем элемент с родителем, если он меньше родителя, свапаем их.
sift-down - если элемент больше детей, сравниваем детей и свапаем элемент с меньшим из них
Для выдачи минимального элемента, обращаемся к корню.
Для вставки - вставляем в конец и просеиваем


Сортировка кучей: HeapSort
1) Put every element in heap
2) Get every element from heap
	O(NlogN)
Для того, чтобы экономить пространство, не создаём второй массив, а формируем кучу на уже обрабатываемом массиве.
Двигаемся слева направо и текущий элемент просеиваем вверх.
Далее достаем по одному первые элементы, свапаем их с последним.
